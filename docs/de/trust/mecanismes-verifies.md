# Verifizierte Mechanismen: Garantie, Funktion, Test

Eine Garantie ist nur dann etwas wert, wenn der Code sie durchsetzt und ein Test sie schützt. Diese Ansicht versammelt die Garantien, die BASE genau auf diese Weise einhält: Jede Zeile verbindet eine Garantie mit der Funktion, die sie durchsetzt, und mit dem Test, der **fehlschlagen** würde, wenn sie brechen würde. Nichts davon ist erfunden. Es ist die kurze Lektüre für ein Bewertungsdossier. Die vollständige Zuordnung von Anforderungen zu Tests liegt in `specs/current/10_core/requirements-matrix.md` und wird in der Continuous Integration neu generiert und überprüft.

Erinnerung an das Vokabular: Ein **Mechanismus** wird vom Broker durchgesetzt (Code prüft ihn, ein Test schützt ihn). Eine **consigne** ist eine Anweisung, die das Modell befolgt, und daher fehlbar. Diese Seite listet nur Mechanismen auf. Die Unterscheidung wird in [`mecanismes-vs-consignes.md`](mecanismes-vs-consignes.md) ausführlich erklärt, und die Verbindung zwischen jeder öffentlichen Aussage und ihrem Beweis in [`evidence.md`](evidence.md).

| Garantie | Mechanismus (Code) | Test, der fehlschlagen würde | Geltungsgrenze |
| --- | --- | --- | --- |
| Eine vertrauliche Ressource gelangt nicht zu einem entfernten Modell, geprüft **vor** dem Aufruf | `tools/core/egress.mjs` → `checkEgress` | `tests/base-egress.test.mjs` (eine geheime IBAN erreicht das Modell nicht) | Vom Broker vermittelte Pfade (MCP-Server, Studio-Chat, Bewertung); ausgelöst durch das Flag `confidential` oder eine `local-only`-Wurzel, nicht durch die Taxonomie `sensitivity`; **die Voreinstellung einer Wurzel ist permissiv (`any`)**, sodass nur eine markierte Ressource (oder eine `local-only`-Wurzel) zurückgehalten wird (siehe [Die Grenze](frontiere-local-vs-sortant.md)); bei direkter CLI-Nutzung bleibt der Mensch die Autorität. |
| Der MCP-Server behandelt einen verbundenen Client standardmässig als **entfernt**: vertrauliche und `local-only`-Wurzeln werden unabhängig vom Transport (`stdio` wie HTTP) zurückgehalten | `mcp/src/base-core-adapter.ts` (egress `modelLocality: remote` als Voreinstellung) | `mcp/tests/index.test.ts` (eine vertrauliche Ressource wird zurückgehalten, ausser bei `BASE_MCP_ALLOW_CONFIDENTIAL=1`, FR-EGRESS-004) | Ein Operator, der weiss, dass der Client lokal und vertrauenswürdig ist, kann ihn mit `BASE_MCP_ALLOW_CONFIDENTIAL=1` zulassen. |
| Ein sensibler Schreibvorgang wird nie ohne Bestätigung ausgeführt | `tools/core/policy.mjs` (Verweigerung eines nicht bestätigten `sensitive`/`restricted`-Schreibvorgangs) | `tests/base-core.test.mjs` (das Opt-out gilt auf diesen Stufen nie) | Schreibvorgänge, die über den Broker laufen. |
| Das Modell schlägt vor, der Mensch validiert: ein zweistufiger Schreibvorgang, vermittelt und atomar | `tools/core/writes.mjs` → `createBrokerWrites` | `tests/base-core.test.mjs` (ein `commitChange` ohne Bestätigung wird verweigert) | Ohne den Validierungsschritt wird nichts geschrieben. |
| Pfade sind eingegrenzt, ausgehende Symlinks werden verweigert, verschachtelte Wurzeln isoliert | `tools/core/confine.mjs` | `tests/base-core.test.mjs` | Schützt den Dateizugriff unter dem Broker. |
| Das routing wählt einen Agenten und einen Prozess oder **enthält sich**, und zwar deterministisch | `tools/core/routing.mjs` → `decideRoute` | `base route-test`, in der CI Wurzel für Wurzel wiederholt (`.github/workflows/ci.yml`) | Lexikalisches routing als Voreinstellung (kein Netzwerk); das semantische Ranking ist optional und kann lokal laufen. |
| Der Kern hat keine Laufzeitabhängigkeit (er läuft mit blossem `node`) | `package.json` (keine Abhängigkeiten) und die Engine `tools/**` | `tests/architecture.test.mjs` (schlägt bei jeder deklarierten Abhängigkeit oder jedem nicht relativen Import fehl) | Das Studio (Webanwendung), der MCP-Server und die Generierung der Doku-Website haben ihre eigenen Abhängigkeiten. |
| Die Dokumentation hat keine toten Links und keine verwaisten Seiten | `tools/docs/model.mjs`, `tools/doctor/diagnose.mjs` | `tests/base-docs.test.mjs`, `tests/base-doctor.test.mjs` | Deckt das Dokumentations-Korpus ab. |
| Das Studio lauscht nur lokal (loopback) | `tools/studio/server.mjs` (Verweigerung jedes nicht-loopback Hosts) | `tests/studio-server.test.mjs` | Das Studio exponiert sich nicht gegenüber dem Netzwerk. |

Jede Zelle «Mechanismus» und «Test» benennt eine reale Datei im Repository: Sie können sie öffnen, die Funktion lesen und den Test ausführen. Genau das bedeutet «durch einen Dritten überprüfbar»: Die Garantie ist kein Versprechen, sondern ein Verhalten, das der Code durchsetzt und ein Test schützt.

Für das, was BASE allein **nicht** garantiert (Richtigkeit der Ausgaben eines Modells, IAM, DLP, gesetzliche Archivierung und der Rest), siehe [`securite-et-limites.md`](securite-et-limites.md). Für die Gerichtsbarkeit des Hosters und die extraterritoriale Exposition siehe [`souverainete-et-confiance.md`](souverainete-et-confiance.md).
