# BASE lernen, indem Sie es tun

**Ihr KI-Werkzeug ist das Erlebnis; Studio ist die Werkstatt.** Dieses Tutorial lässt Sie mit
eigenen Händen einen Assistenten für ein fiktives Tourismusbüro (**Veytaux-les-Bains**) bauen und
ihn anschliessend auf IHRE Dateien bringen. Jeder Schritt endet mit einer Überprüfung: Sie gehen
nie weiter, ohne den Beweis, dass alles funktioniert. Das ist der Vorteil von BASE: Jeder Schritt
wird durch eine deterministische Kontrolle validiert, die Sie prüfen und bestätigen.

## Wählen Sie Ihren Ausgangspunkt

### Entdeckung: 32 Min, auf der BASE-Seite nichts zu installieren
Sie haben noch nie die Arbeit einer KI strukturiert. Sie öffnen das bereits fertige Tourismusbüro,
sprechen mit ihm, ändern eine Regel und sehen, wie der Assistent der Datei gehorcht. **3 Module.**
Am Ende: Sie haben Ihren eigenen Ordner und eines Ihrer echten Dokumente darin.
→ [Schritt 0: Ihr Werkzeug anschliessen](harnais.md) dann [Modul 1](decouverte-1-faites-le-parler.md).

### Praktiker: 122 Min, Node installiert
Sie wollen bauen. Sie erstellen Ihr echtes Projekt mit `base init`, bauen das Tourismusbüro aus
Lückengerüsten neu auf, fügen Kompetenzen und Vorlagen hinzu, um Dokumente zu generieren, bewerten
die Qualität, bearbeiten eine Rückmeldung aus der Praxis und migrieren dann Ihre eigenen
Verfahren. **9 Module.**
Am Ende: IHR Assistent antwortet auf IHREN Inhalt und bereitet Ihre Liefergegenstände vor.
→ [Schritt 0](harnais.md) dann [Modul 1](praticien-1-anatomie.md).

### Team: 50 Min
Sie stellen für mehrere Personen bereit. Workspace mit mehreren Geltungsbereichen, Egress-Governance
über eine echte vertrauliche Ressource, Verteilung per git und MCP-Server. **3 Module.**
→ [Schritt 0](harnais.md) dann [Modul 1](equipe-1-workspace.md).
