# Die erste Evaluation

*⏱ ~15 Min · Modul 7/9, Parcours Praktiker*

**Sie werden**: das Urteil eines Richters zu einem Szenario lesen und einen Ansatz zur Verbesserung erkennen, belegt durch das ✅ weiter unten.
**Sie brauchen**: Modul 6 abgeschlossen (ein verbundenes Modell, Standardvorgaben für die Evaluation).
↻ **Erinnerung**: Ohne nachzuschauen, warum stehen die API-Keys nie im Bildschirm? (sie leben in einer Umgebungsvariablen, nie in den Dateien)

1. In Studio, Tab **Evaluationen**, Schaltfläche «▶ Evaluieren».
2. Das Panel öffnet sich, vorausgefüllt. Das Tourismusbüro von Veytaux liefert zwei Szenarien (ein
   Durchreisender, vom Wetter enttäuscht, eine vage Gruppenverantwortliche). Starten Sie.
3. Warten Sie: Ein simulierter Benutzer spielt das Szenario durch, dann bewertet ein unabhängiger Richter jede
   Konversation. Die Ergebniskarten treffen eine nach der anderen ein.
4. Klicken Sie auf eine Karte, um das Urteil des Richters und seinen Korrekturhinweis (fix hint) auszuklappen.

✅ **Prüfen Sie**: Sie sehen mindestens ein Urteil (erreicht / teilweise / verfehlt) mit einer Begründung und, falls das Ziel nicht erreicht ist, einen konkreten Korrekturhinweis (fix hint).

💡 **Warum es funktioniert hat**: Evaluieren heisst nicht «der Code kompiliert»: Es ist ein echter simulierter Benutzer, der einem echten Richter gegenübersteht, auf Szenarien, die Sie schreiben. Das Urteil sagt Ihnen, ob der Prozess hält, BEVOR Ihre echten Besucher kommen.

🔁 **Bei Ihnen**: Welches Fallen-Szenario möchten Sie, dass Ihr Assistent garantiert besteht (der Besucher, der eine Information vergisst, die Grenzanfrage)?

→ **Und jetzt**: [Modul 8: das Feld](praticien-8-le-terrain.md): wenn eine echte Nutzung ein Problem zutage fördert.

🆘 **Häufige Pannen**: *Die Evaluation scheitert beim Start*: Eine Meldung listet die Probleme (Provider/Modell) auf und verweist Sie zurück auf die Einstellungen. *Es ist langsam*: Ein lokales Modell braucht ein paar Minuten, das ist normal.
