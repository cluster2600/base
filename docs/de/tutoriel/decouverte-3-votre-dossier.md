# Ihr Ordner, Ihr erstes Dokument

*⏱ ~12 Min · Modul 3/3, Parcours Entdeckung*

**Sie werden**: einen eigenen BASE Bereich mit einem Ihrer echten Dokumente darin haben und wissen, dass es wirklich gelesen wird, bewiesen durch das ✅ weiter unten.
**Sie brauchen**: die Module 1-2 abgeschlossen.
↻ **Erinnerung**: Was verändert, ohne nachzuschauen, das Verhalten des Assistenten? (seine Dateien bearbeiten)

1. Kopieren Sie den Ordner `exemples/starter-perso` aus dem Repository heraus (in Ihre Dokumente) und
   benennen Sie ihn um, zum Beispiel `mon-assistant`. Mit der Tastatur: Rechtsklick -> Kopieren -> Einfügen. Oder
   bitten Sie Ihren Assistenten: *«kopiere den Ordner starter-perso in meine Dokumente»*.
2. Öffnen Sie DIESEN neuen Ordner in Ihrem Werkzeug (*File -> Open Folder*).
3. Ziehen Sie eines Ihrer echten Dokumente hinein (eine Prozedur, ein Merkblatt). Sichere Formate: `.md`, `.txt`.
   Falls das Format nicht funktioniert, fügen Sie den Text in eine neue `.md` Datei ein.
4. Stellen Sie drei Dinge (das **Befragungsprotokoll**):
   - eine Frage, die NUR Ihr Dokument beantworten kann (es muss das Dokument zitieren);
   - eine Frage ausserhalb Ihres Dokuments (es muss sagen, dass es nicht Bescheid weiss);
   - eine Anfrage, die zu einer Ihrer Aufgaben routen sollte.

✅ **Prüfen Sie**: Antwort 1 zitiert Ihr Dokument; Antwort 2 gibt zu, dass sie es nicht weiss, statt zu erfinden; Antwort 3 trifft die richtige Absicht. Diese drei zusammen beweisen, dass der Import echt ist und dass der Assistent nicht blufft.

💡 **Warum es funktioniert hat**: ein erfolgreicher Import ist nicht «die Datei ist da», sondern «der Assistent nutzt sie und bleibt ehrlich über seine Grenzen». Das Befragungsprotokoll testet genau das, ganz ohne Werkzeuge.

🔁 **Bei Ihnen**: Welches Ihrer Dokumente verdient es, der erste echte Inhalt Ihres Assistenten zu sein?

→ **Und jetzt**: Sie haben die Entdeckungstour abgeschlossen. Um wirklich zu bauen (Prozesse, Evaluation, Praxis), gehen Sie weiter zum [Parcours Praktiker](praticien-1-anatomie.md).

🆘 **Häufige Pannen**: *Er erfindet eine Antwort auf Frage 2*: das ist das Risiko der Consigne Stufe; das deterministische Routing und ein gutes use_when verringern es (Parcours Praktiker). *Er sieht Ihr Dokument nicht*: prüfen Sie, dass es wirklich IM geöffneten Ordner liegt, nicht woanders.
