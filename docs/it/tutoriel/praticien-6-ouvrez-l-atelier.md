# Apri l'officina, collega un modello

*⏱ ~10 min · modulo 6/9, percorso Praticante*

**Otterrai**: l'officina aperta e un modello collegato, pronto per la valutazione, dimostrato dal ✅ qui sotto.
**Ti serve**: il modulo 5 completato.
↻ **Ripasso**: senza guardare, cosa fa base doctor? (segnala ciò che sta per rompersi)

1. Avvia l'officina: `base studio --root exemples/veytaux-tourisme`. Il tuo browser si apre.
2. Scheda **Impostazioni**. Se non c'è alcun provider, appare la guida «Collega un modello».
3. Segui il percorso più semplice per te (Ollama locale senza key, oppure una key di API che
   incolli nel tuo terminale). La guida stampa ogni volta il passo successivo.
4. Una volta che il test di connessione è verde, clicca «imposta come predefiniti di valutazione».

✅ **Verifica**: la schermata Impostazioni mostra il tuo provider con un test di connessione verde, e i tuoi predefiniti di valutazione sono impostati.

💡 **Perché ha funzionato**: Studio è l'officina, lavora sugli stessi file del tuo strumento IA. Le key di API non vengono mai digitate né memorizzate nella schermata: dai un nome a una variabile d'ambiente, il che mantiene i tuoi segreti fuori dai file.

🔁 **A casa tua**: quale modello userai per le tue valutazioni, un modello locale (gratuito, privato) o un'API (più potente)?

→ **E adesso**: [Modulo 7: la prima valutazione](praticien-7-premiere-evaluation.md).

🆘 **Guasti comuni**: *«Nessun modello»* dopo aver installato Ollama: ricarica la pagina delle Impostazioni. *La key non viene rilevata*: esporta la variabile nello stesso terminale di `base studio`, poi rilancia.
